这是主程序目录。完整项目介绍请查看仓库根目录 README。

## 启动

    cd yolo_action_detection
    ..\.venv\Scripts\python.exe src\main.py

## 打包

    cd yolo_action_detection
    .\packaging\build_yolo.bat

## PCB 多板元器件检查模式

当产线需要同时检查多块 PCB 上的元器件是否齐全时，启用 PCB 检查模式。

### 模型类别约束

模型必须在一个完整画面中同时输出以下类别：

- **PCB 类别**：如 `pcb`，用于跟踪产品身份
- **四个元器件类别**：如 `R1`、`C2`、`U3`、`Q4`，名称互不重复且不与 PCB 类别相同

### 使用条件

- 多块 PCB 沿同一流水线运动并**保持间隔**，不得相互接触或重叠
- PCB 必须**完整进入画面安全区域**后才开始检查，部分入画的 PCB 保持等待
- 每个画面只执行一次模型推理，同时更新所有可见 PCB 的状态

### 判定规则

- **PASS**：一个 PCB 在有效检查帧中同时检测到全部四个元器件 -> 立即 PASS
- **FAIL**：一个 PCB 连续 **3 个有效帧**（默认值）均缺少至少一个元器件 -> FAIL
- 第三帧恢复完整 -> PASS，不判 FAIL
- 已完成判定（PASS 或 FAIL）的 PCB 不再重复检查或计数

### 配置方法

在配置页 → 高级配置 → PCB 多板元器件检查：

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| 启用 PCB 检查模式 | 关闭 | 开启后切换到 PCB 模式，关闭则使用动作顺序模式 |
| PCB 类别 | pcb | 模型中 PCB 的类别名称 |
| 元器件 1-4 类别 | 空 | 四个互不重复的元器件类别名，留空会拒绝启用 |
| FAIL 连续帧 | 3 | 连续多少个有效帧缺件才判 FAIL |
| 轮次间隔(秒) | 0.0 | 结果输出之间的最小间隔，0 表示不等待 |
| 归属容差 | 0.15 | PCB 归属区域扩张比例，用于容纳板边元器件 |

### 现场调参方法

1. **元器件归属不准**：调整归属容差。板边元器件中心在 PCB 外时，增大容差（如 0.2-0.3）；相邻 PCB 误吸附时，减小容差（如 0.1）
2. **FAIL 太快/太慢**：调整 FAIL 连续帧。默认 3 帧，产线速度慢可增到 4-5 帧，速度快可降到 2 帧
3. **结果输出太快**：调整轮次间隔。设为 1-2 秒可以让操作员有足够时间看到每次结果
4. **PCB 不被检查**：确认 PCB 完整在画面安全区域内，接触边界的 PCB 不会累计 FAIL

### 画面标注

PCB 模式下，画面上会显示：
- 每个 PCB 上方标注 `PCB #ID` 和状态（PASS/NG/检测中）
- 检测中的 PCB 显示已识别槽位数和连续 FAIL 帧数（如 `3/4 FAIL:1`）
- FAIL 的 PCB 显示缺失的元器件类别
- 每个已归属的元器件旁标注所属 PCB 的 ID
