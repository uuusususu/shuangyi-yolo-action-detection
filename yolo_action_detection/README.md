## 界面与配置

主界面采用“顶部状态栏 + 左侧相机画面 + 右侧生产信息”的布局。普通顺序模式按当前轮次显示已识别步骤；首类别区域检查模式只显示当前轮次父类 ID 的步骤，并展示数量进度，例如 `脚垫 3 / 4` 或 `脚垫 0 / 4`。旧 FAIL 父类可以继续在画面中红框显示，但不会覆盖当前 ID 的步骤卡。

标准按钮、输入框、下拉框、数字框和复选框由 MIT 许可的 `PyQtDarkTheme 2.1.0` 统一绘制边界、箭头和交互状态；页面布局、深蓝/青色科技风以及 PASS/FAIL 状态色仍由项目 UI token 管理。主题依赖属于便携版运行依赖，源码环境缺失时会明确报错，打包冒烟测试也会检查主题模块是否存在。

配置页按以下六个分区组织，切换分区不会丢失尚未保存的输入值：

- 模型与步骤：模型、推理参数、类别顺序和同帧目标数量
- 动作判定：PASS/NG 稳定帧、顺序约束和完成一轮间隔
- 工业相机：在线设备选择、刷新、参数模式、参数组、参数文件和曝光
- 显示与反馈：置信度、Track ID、PASS 提示音、FAIL 提示音和失败证据
- 区域检查：首类别区域检查开关、FAIL 稳定帧和归属边距
- 生产统计：目标产能、当前批次统计和归零归档

“完成一轮间隔”统一保存到 `round_cooldown_seconds`，普通顺序检测和首类别区域检查都使用该值。首类别模式下，当前父类 ID PASS 或 FAIL 后会立即预选下一个等待 ID，但只有间隔结束后才开始统计新 ID 的子类别数量；间隔为 0 时立即开始。

## 迈德工业相机选择

程序每次启动都会通过 MvSDK 重新枚举当前在线的迈德相机。设备目录只保存在本次运行的内存中，不会把枚举列表或枚举索引写入配置文件。

在配置页“工业相机”分区使用设备下拉框选择相机。每项显示 `FriendlyName | PortType | SN`，可点击“刷新设备”重新获取当前在线列表。配置只持久化所选设备的 `mvsdk_camera_sn`：

- 已保存 SN 仍在线时，启动后默认选中该设备
- 已保存 SN 不在线时，显示离线提示并要求重新选择，不自动改选第一台
- 设备缺少 SN 时只用于诊断显示，不能保存为运行设备
- 旧配置只有 `mvsdk_friendly_name` 时，仅在名称唯一匹配一台在线设备时预选；同名设备必须人工按 SN 选择
- 首次升级且尚未保存 SN 时，必须在此分区明确选择一台相机并保存

设备选择只有点击“保存”后才会应用。相机尚未打开时只保存 SN，不自动打开；当前相机 SN 未变化时保持采集，不重启；SN 变化时停止检测、关闭旧相机并清空旧画面和本轮状态，再打开新相机进入预览。新相机不在线、被占用或初始化失败时保持相机关闭和画面空白，并显示目标 SN 与 SDK 错误原因，不会回退到旧相机或第一台设备。切换和关闭相机不会清零生产统计。

## 首类别区域检查模式

当产线已在“动作类别序列”里按顺序配置类别，且第一个类别代表产品主体、工装区域或父对象时，启用首类别区域检查模式：系统自动使用类别 1 作为父区域，使用后续非空类别作为需要检查是否位于父区域内的子控件。

示例配置：

| 步骤 | 类别 | 含义 |
| --- | --- | --- |
| 类别 1 | 产品主体 | 父区域 |
| 类别 2 | 脚垫 | 子控件 |
| 类别 3 | 螺丝 | 子控件 |
| 类别 4 | 标签 | 子控件 |

启用后，系统每帧先识别“产品主体”并为每个实例建立业务追踪 ID，再分别统计唯一归属于该主体区域的脚垫、螺丝、标签数量。某个子类别连续达到正确配置数量后，系统把该步骤锁存为完成；后续短暂漏检不会撤销。不同子步骤可以在不同帧完成，不要求所有类别同帧出现。

### 模型类别约束

模型必须在一个完整画面中同时输出以下类别：

- **父区域类别**：动作类别序列的类别 1，用于定位产品主体或控件区域
- **子控件类别**：动作类别序列中类别 2 及之后的非空类别，用于检查是否位于父区域内

### 使用条件

- 第一个配置类别必须能检测到完整父区域，例如产品主体、工装区域或控件区域
- 子控件必须位于父区域内，父区域之间应保持间隔，不得相互接触或重叠
- 父区域必须完整进入画面安全区域后才开始检查，部分入画时保持等待
- 每个画面只执行一次模型推理，同时追踪所有可见父区域；同一时间只有一个当前父类 ID 推进子类别数量判定

### 静态图片验收

在 `yolo_action_detection` 目录运行：

```powershell
..\.venv\Scripts\python.exe src\validate_onnx_picture.py `
  --image test.jpg `
  --model config\sanrepian2.onnx `
  --region-config config\config.json `
  --output-dir outputs\first_category_test_validation `
  --conf 0.65 --iou 0.5
```

输出 JSON 中的 `first_category_region` 会列出父类类别、父类业务 ID、每个子类别的区域内实际数量、配置数量和最终状态。只有至少识别到一个父类，且当前需判定的父类 ID 都按稳定帧规则完成 PASS 时命令才退出 `0`；`no_parent`、`waiting` 和 `ng` 都会返回非零退出码。

当前验收样本 `test.jpg` 使用 `sanrepian2.onnx` 和生产配置运行时，先识别到 1 个 `pcb` 父类并分配业务 ID `-1`，再统计该父类区域内 4 个 `sanreidian`，最终状态为 `pass_latched`，只输出一次 PASS。

### 判定规则

- **等待/冷却**：父类 ID 尚未轮到当前判定，或 PASS/FAIL 后正在等待 `round_cooldown_seconds` 间隔时，只追踪和显示父类，不统计子类别
- **数量异常**：当前轮次父类 ID 的未完成子步骤只要数量不等于配置值就累计 FAIL 连续帧，`0/4` 也会累计
- **步骤完成**：子类别在父区域内连续达到正确配置数量并满足 PASS 稳定帧后锁存；后续漏检不撤销
- **PASS**：同一父类 ID 的所有子步骤都已锁存完成后，只输出一次 PASS；PASS 才是该 ID 的终局完成状态
- **FAIL**：未完成子步骤数量连续达到配置的 NG 稳定帧数仍不等于配置数量时，父类产生本轮 FAIL/NG 报警并排回等待队列
- **红框可恢复**：同一父类 ID FAIL 后只要当前帧仍识别到就绘制红色父类框；后续补齐配件并 PASS 后恢复为完成颜色；当前帧识别不到时不绘制陈旧红框
- **事件去重**：声音按识别轮次去重，每个完成的 PASS 或 FAIL 轮次最多播放一次提示音；同一父类 ID 重新入队后的新轮次再次 FAIL 时会再次播放。统计与证据仍按异常签名去重，不受声音轮次规则影响

### 声音反馈

系统提供两个独立的声音开关，互不影响：

| 开关 | 默认值 | 说明 |
|------|--------|------|
| PASS 提示音 | 关闭 | 每个完成的 PASS 结果播放一次通过提示音 |
| FAIL 提示音 | 开启 | 每个完成的 FAIL 轮次播放一次异常提示音 |

- 旧配置只有 `sound_feedback_enabled` 时，自动迁移为 FAIL 提示音开关值，PASS 提示音保持关闭
- 配置保存后立即生效，无需重启程序或重新打开相机
- 声音播放失败（如资源缺失）不影响检测、统计和画面显示
- FAIL 后会立即开始调度下一个等待 ID；旧红框不影响下一个当前 ID 的步骤卡。已 PASS 的父类 ID 不再重复检查、反馈或计数

### 配置方法

在“模型与步骤”分区先配置类别顺序，再到“区域检查”分区启用首类别区域检查：

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| 启用首类别区域检查 | 关闭 | 开启后类别 1 作为父区域，类别 2 及之后作为子控件 |
| 类别 1 | 空/默认步骤 | 父区域类别，例如产品主体 |
| 类别 2-N | 空/默认步骤 | 子控件类别，例如脚垫、螺丝、标签 |
| 归属容差 | 0.15 | 父区域归属范围扩张比例，用于容纳边缘子控件 |

### 现场调参方法

1. **子控件归属不准**：调整归属容差。边缘子控件中心在父区域外时，增大容差（如 0.2-0.3）；相邻父区域误吸附时，减小容差（如 0.1）
2. **数量异常锁存太快/太慢**：调整“动作判定”里的 NG 稳定帧数。首类别模式直接复用该值，当前建议 10 帧；仍然误检抖动时可增到 15-20 帧
3. **步骤完成太快/太慢**：调整动作判定中的 PASS 稳定帧数，该值同时用于首类别模式的正确数量稳定判断
4. **父区域不被检查**：确认父区域完整在画面安全区域内，接触边界的父区域不会推进稳定计数

### 画面标注

首类别区域检查模式下，画面上会显示：
- 当前帧中的每个父类都标注独立业务 ID 和状态（PASS/NG/检测中/等待中）
- 只有当前轮次父类 ID 会推进步骤数量，步骤卡与顶部状态绑定同一个 ID
- FAIL 可重试的父类当前可见时为红色，PASS 为绿色，观察中为橙色
- 右侧步骤详情明确绑定一个父类 ID，不合并多个父类的子类别数量
